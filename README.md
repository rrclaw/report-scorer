# report-scorer

一个用于 [Claude Code](https://docs.claude.com/en/docs/claude-code) / Claude Agent 的 **标准化研报评分 skill**。用一套固定、行业无关的 rubric，给任意一份投研/行业研究报告打**公允、可复现**的分数。

> 核心价值：**同一套标准，换谁评、评哪个行业，都给一致的分数。** 不当舔狗——再好看的报告，可追溯性差就扣到位；再朴素的 wiki，证据扎实就给高分。

## 为什么需要它

研报质量判断天然主观："今天评 90、明天评 70""评自家手松、评对手手紧"是常态。本 skill 用四个机制把主观判断机械化：

1. **锚定式评分带**——每个维度 5 档，每档有具体文字标准，拿证据对档，不凭印象。
2. **证据绑定**——打分前必须摘 2-3 条原文证据（引文+位置），没证据不能给高分。
3. **硬性扣分台账**——无 source / 陈旧未标 / 单源决策数字 / 现货价线性外推 / 单边无反方……逐条扣，可对账。
4. **行业自适应清单**——评"完整性"前先载入该行业应有要素清单（化工看成本曲线/价差，消费看品牌/渠道/同店，科技看技术代际/卡脖子），不拿一把尺子套所有行业。
5. **校准锚**——4 个已知分数的标杆样例，评分前对齐尺子。

## 七维 rubric（加权合 100）

| 维度 | 权重 | 考察 |
|---|---:|---|
| 可追溯性 | 20 | 每个材料级结论是否挂得住来源、分一手/二手、标置信度与日期 |
| 逻辑严谨性 | 20 | 结论是否从证据推得出、可证伪、不过度外推、决策数字三角验证 |
| 深度与洞察 | 18 | 有无超出共识的预期差、二阶效应、量化机制、反方论证 |
| 投资有效性 | 15 | 能否直接决策：分层标的、空间/赔率、进出条件、证伪触发 |
| 系统完整性 | 12 | 是否覆盖该行业应有分析骨架，无重大盲区 |
| 时效性 | 10 | 数据新不新、陈旧是否标注/归档、有无变更追踪 |
| 可读性与结构 | 5 | 是否分层、可导航、信息密度高 |

> 权重逻辑：可追溯 + 逻辑 = 40 分压舱，因为信不过的报告再深也是负资产；可读性只占 5 分，好看不能替代对错。

## 安装

把整个目录放进你的 Claude skills 目录：

```bash
git clone https://github.com/rrclaw/report-scorer.git ~/.claude/skills/report-scorer
```

（Claude Code 会自动发现 `~/.claude/skills/` 下的 skill。）

## 用法

直接对 Claude 说，比如：

- "给这份研报打分" + 报告文件/粘贴全文
- "这份行业分析能打几分"
- "评估一下这份券商深度的质量"
- "用 rubric 评一下这份报告"
- "对比这两份报告谁更好"

输出是一张**记分卡**：维度得分 + 证据 + 扣分 + 加权总分 + 等级 + Top5 优点 + Top5 短板（按修复杠杆排）+ "离 100 差什么"的行动清单。

支持两种模式：**单遍**（快）和**三遍独立评分取均值**（降方差，用于高 stakes 场景）。

## 等级

| 等级 | 总分 | 含义 |
|---|---|---|
| S | 90+ | 卓越机构级，挑不出实质毛病 |
| A | 80-89 | 优秀，标准动作做全 |
| B | 70-79 | 及格偏上，有明显短板 |
| C | 55-69 | 好看但信不过 / 缺斤短两 |
| D | <55 | 不具备作为决策依据的可靠度 |

## 结构

```
report-scorer/
├── SKILL.md                          # 协议层：七维权重 + 六步流程 + 公允性铁律
└── references/
    ├── rubric.md                     # 尺子主体：七维 × 五档锚定描述
    ├── deduction_ledger.md           # 硬性扣分台账 + 红线 BLOCK
    ├── industry_checklists.md        # 7 套行业应有要素清单
    ├── calibration_anchors.md        # 4 个已知分数的标杆样例
    └── output_template.md            # 记分卡输出格式
```

## 自定义

- **改权重**：编辑 `SKILL.md` 七维表 + `output_template.md`。
- **加行业清单**：在 `industry_checklists.md` 加一套，evaluation 会自动选最贴近的。
- **改合规红线**：`deduction_ledger.md` 末尾的红线清单是模板，按你的发布平台/司法辖区增删。
